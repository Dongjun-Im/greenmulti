# 초록멀티 v1.8 — 게시판 조회수·접근성·로그인 다이얼로그 개편

## 새 기능

### 1. 게시판 목록·게시물 본문에 조회수 표시
- **게시판 목록** 항목에 `[조회 N]` 표시 — 지금까지 보이지 않던 조회수가
  댓글 수와 함께 노출됩니다.
- **게시글 정보 영역** 과 본문 헤더에도 `조회수: N` 표시.
- **첨부파일 목록** 에 `(다운로드 N회)` 표시.
- 게시판 목록에서 항목 선택 후 **Shift + 오른쪽 방향키** 로 필드 순회 시
  조회수도 함께 읽어줍니다 (번호 → 제목 → 작성자 → 날짜 → 댓글 → 조회수).
- gnuboard5 의 `td_num` 두 번째 셀, 본문 정보 영역의 `"N읽음"` 등 소리샘
  스킨이 쓰는 다양한 표기 모두 지원.

### 2. 게시물 본문에서 N / P 키로 이전·다음 게시물 이동
- **N 키**: 다음 게시물
- **P 키**: 이전 게시물
- 기존의 `Alt+B` (이전) / `Alt+N` (다음) 도 그대로 사용 가능.
- 댓글 목록에 포커스가 있을 때 **N** 은 댓글 정렬 토글로 동작 (충돌 방지).

## 버그 수정

### 게시판 / 게시물

1. **상단 공지에서 이전·다음 게시물이 엉뚱한 글로 점프하던 문제 해결**
   - gnuboard 가 공지(sticky) 게시물의 "이전글/다음글" 링크를 다른 공지 또는
     예측 불가능한 글로 연결하는 문제. 이제 화면에 표시되는 게시판 목록
     순서를 기준으로 한 칸 이동.
   - 첫 게시물에서 이전 → "첫 게시물입니다." 안내, 이동 안 함.
   - 마지막 게시물에서 다음 → "마지막 게시물입니다." 안내, 이동 안 함.
   - 안내 음성이 다이얼로그 깜빡임에 잘리지 않도록 PostDialog 내부에서 처리.

2. **게시물 제목에서 "v1.7" 이 "v.7" 처럼 깨져 보이던 문제 해결**
   - gnuboard5 가 제목 `<a>` 태그 안쪽에 `<span class="cnt_cmt">N</span>` 을
     끼워 넣는데, 이를 단순 `replace` 로 제거하다가 제목 본문의 같은 숫자가
     함께 잘려나가던 문제 (`v1.7` → `v.7`). 링크 사본을 만들어 sound_only /
     cnt_cmt 노드만 제거 후 텍스트 추출하도록 수정.

3. **공지 첫 게시물에서 P 또는 Alt+B 시 안내 음성이 끊기던 문제 해결**
   - 경계 안내 직후 PostDialog 가 재생성되며 스크린리더가 제목을 다시 읽어
     안내가 잘리던 증상. 다이얼로그를 새로 만들지 않고 같은 다이얼로그를
     유지한 채 안내만 출력.

4. **메인 메뉴 개수 음성 안내 정확도 수정**
   - 자료실/전자도서관/노원시각장애인학습지원센터 등 보강 메뉴 포함한 16개
     기준으로 안내 (이전엔 raw 13개로 잘못 안내).

### 인증 / 로그인

5. **인증 음성 안내 ("인증 중입니다", "초록멀티를 실행합니다") 가 들리지
   않던 문제 해결**
   - 다음 단계 발화가 즉시 따라붙어 PurgeBeforeSpeak 으로 잘리던 현상.
   - `green_auth.screen_reader.speak()` 에 발화 길이 추정 기반 wx-yield
     대기 추가. wx 이벤트 루프는 살려두고 발화 끝까지 대기.
   - 진행 인디케이터의 "인증 진행 중" 발화는 시작 후 6 초간 보류해 초기
     인증 안내가 묻히지 않게 보호.
   - 메뉴 자동 감지 시작 시의 중복 발화 ("메뉴를 불러오는 중입니다") 제거.

6. **로그인 다이얼로그 비밀번호 마스크가 "괄호닫고" / "짭짭" 처럼 깨지던
   문제 해결**
   - OS 기본 마스크 문자(●/•) 를 일부 한국어 스크린리더(센스리더 등) 가
     깨진 발음으로 읽어주던 문제. 수동 마스킹으로 전환 — 화면에는 `*` 만
     표시, 실제 비밀번호는 별도 멤버에 보관. 모든 한국어 리더에서 안정적으로
     "별표" 로 발음.
   - **녹색등대 인증 패키지** 와 **NAS 로그인 다이얼로그** 모두 동일 적용.

7. **Tab / Shift+Tab 시 포커스 안내가 두 번 눌러야 들리던 문제 해결**
   - 기존 `wx.CallLater(200ms)` 지연 안내가 연타 시 다음 포커스에 묻혀 첫
     안내가 사라지던 증상. 모든 컨트롤(아이디·비밀번호·체크상자) 포커스
     이벤트를 즉시 안내로 변경.
   - 다이얼로그 첫 자동 포커스만 한 번 건너뛰어 스크린리더가 다이얼로그
     제목 ("초록등대 인증") 을 끝까지 읽도록 보호.

8. **체크박스 라벨 통일** — "아이디와 비밀번호 저장" → "아이디/비밀번호 저장".
   `name` 속성도 "아이디/비밀번호 저장 체크상자" 로 명시.

9. **NAS 로그아웃 다이얼로그 문구** — "아이디·비밀번호" 가운뎃점 표기를
   "아이디/비밀번호" 슬래시 표기로 통일.

### 스크린리더 호환성

10. **센스리더가 늦게 켜졌을 때 영영 발화 안 되던 문제 해결**
    - `_SENSE_APP` COM 객체가 모듈 import 시점에만 한 번 캐싱되어, 사용자가
      초록멀티 실행 후 센스리더를 켜면 영영 인식되지 않던 증상. lazy 로딩으로
      변경 + 발화 실패 시 캐시 클리어로 다음 호출 시 재시도.

11. **센스리더가 자동 감지 우선순위에서 밀려 SAPI 폴백으로 잡히던 문제 해결**
    - `accessible_output2` 의 `Auto()` 가 활성 리더 없을 때 SAPI 로 silently
      폴백하면서 True 를 반환해 센스리더가 영영 호출되지 않던 증상. 발화
      체인을 재배치: VoiceWith → ao2_nvda → nvda_direct → **센스리더 (앞당김)**
      → ao2_auto → SAPI.

## 동기화

`green_auth/` 패키지의 위 변경 사항을 **업스트림 / 초록멀티 / 동호회 회원관리**
세 위치 모두에 동기화. 회원관리 프로그램은 별도로 빌드(`pyinstaller
chorok_green_admin.spec`) 해야 합니다.

## 업그레이드 안내

- 기존 v1.7.x 사용자는 이 인스톨러 실행만으로 자동 업그레이드 됩니다.
- 시작 메뉴 / 바탕화면 바로가기는 "초록멀티" 단일 이름으로 유지되며 v1.7
  잔해(`초록멀티 v1.7.exe`, `초록멀티 v1.7.lnk` 등) 는 설치 시 자동 정리.
- 사용자 데이터(메뉴 편집·자격증명·즐겨찾기·테마 설정) 는 그대로 보존.
